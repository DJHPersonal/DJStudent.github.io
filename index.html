Island Survival

Unreal Engine 4 C++ Blueprints Multiplayer Procedural Terrain Student Work(Group of 2)

What I implemented

  • Procedural Island Generation
  • Biomes Generation
  • Plant/Structure Spawning
  • Fuel pickup system
  • LAN Multiplayer

Project Overview

Created as a second year project, where any AI or PCG system could be implemented, this tech demo has players collect fuel for their boat and evade zombies on some randomly generated islands. The terrain system was implemented with the idea of its future use within many other projects so that their development times can be significantly reduced. Due to this all elements have been designed to be highly customisable and easily removable based on the projects needs. The added challenge of island generation was implemented to allow for a more natural border to the game world.

Project selected for and shown at the:

Procedural Island Generation

Using Perlin noise as the base each vertex of the plane has been adjusted with its own height value, having evolved along the following steps:

Added Muliple layers of perlin noise

Gave the terrain more randomness and less repetitivness through domain warping

Made the terrain flatter by adding in more rolling hills

Allowed ridges to appear

Made terrain consist of flatter areas broken up by ridges and gullies

Added option for the terrain to be more stylized through terraces

Added islands by raising up the centre and drastically lowering the edges of the previous map

Biome Generation

Starting with the base generation above, the biome and plant spawning was overlayed on the terrain and evolved along the following steps:

Inital island generation through temperature and moisture noise maps

Overhall of generation to now work so each island has its own individual voronoi noise map

Updated each biomes stats and added basic plant spawning based on terrains vertices

Each biome given own unique generation using method outlined above as well as made plant spawns more realistic

Each biome has its own blueprint holding its stats making it easy to add and remove any biome

Each plant has a density within a biome as well as population which spawn around a random centre point

The Raindrop

Unity v2020.1.1.1.1 C# Gameplay AI Student Work(Group of 5)

What I Implemented

  • Player Controller
  • Random Level Generation
  • All Menus and UI Elements
  • Saving / Loading using PlayerPrefs
  • Wind and Blackhole Systems

Project Overview

This second year project, developed largely between August 2020 - October 2020(10 Weeks) was undertaken to learn the fundamentals of game development. Tasked with recreating the entirety of the game Pac-Man with a distinctive style as well as creating a unique design innovation. Here I challenged myself by implementing an entire procedural maze system, which works around the constraints of the Pac-Man level design.

Project selected for and shown at the:

Procedural Maze Generation

Steps:

  1. Generates maze border
  2. Spawns in centre piece and teleports
  3. Uses a recursive backtracking method to generate the maze paths
    4. This was a particularly challenging system to implement as all inside walls needed to be at least 2-thick requiring a number of extra conditions to be determined and checked against.
  4. Spawn in correct map tiles with appropriate orientation

Sheep AI

Moves Away from Player

Moves Towards Player

Moves in Random Directions

Follows Map Border in Clockwise Direction

One notable challenge here was the 4th Sheep as the waypoint system used had to be easily implementable within the maze generation system. This was solved by making each green tile act as a waypoint with the sheep mealy finding and navigating between each in turn, in a clockwise direction. Accurate navigation was also a substantial challenge, especially on a large map so an efficient A* pathfinding system was implemented to give accurate results. This was achieved by making the green border tiles themselves act as the waypoint system, with them individually storing the neighbours tile location, within a clockwise direction.
